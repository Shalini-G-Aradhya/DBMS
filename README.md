# CodeJudge Database Design - Part 1

This project contains relational database design and normalization for the CodeJudge dataset.

Contents:

* Schema understanding
* Entity identification
* Keys and constraints
* Normalization reasoning
* SQL schema
* ERD diagram

The database is designed using relational DBMS concepts including:

* Primary Keys
* Foreign Keys
* Composite Keys
* Candidate Keys
* Normalization (1NF, 2NF, 3NF)

The schema is designed from raw CSV files provided in the dataset.
